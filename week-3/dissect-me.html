Everything You've Ever Wanted to Know About Version Control

(but were afraid to ask)

October 1st, 2015

"Dude, suckin' at something is the first step to being sorta good at something." Jake the Dog, from the catoon show "Adventure Time".

Hi! My name is Clinton Weber. I am happy that you are here to learn a few things about web development. Before we start diving into the technical jargon, here's a little bit about myself. I have recently enrolled in a program called Dev Bootcamp (DBC). DBC is a 19-week extensive program for folks who want to learn everything they can about what it takes to be a full-stack web developer. So as I write these blog posts, you can rest assure that I have just learned of these tidbits of information myself.

The question, "What is version control?", is something that's asked of us Island Foxes quite often; maybe for a good week now. I realize now, though, that this is a very important question! To fully analyze this quandry, or what I know of it, I must break a form of version control called "GitHub" into two parts: One is Git, a local user interface, and the other is GitHub, a cloud-based collaborative remote work space (www.github.com). These two components are integral to using GitHub to it's full potential. Here is a resource for installing Git: here.. After you do this sucessfully, go to here and create a profile.

Still following? Great. Based on your sheer determination and will, you will learn to be a developer with me. Now that we are all set up, let's rock. Go to your profile on GitHub.com and select the plus sign next to your avatar. Click "New Repository." Fill in the required information, and consider whether you want this project to be opensource, or some other form of creative licensing. When your repository is created, look for the "HTTPS clone URL" on the right-hand side of the displayed repository. Copy-and-paste the URL and open Terminal (or whatever shell you prefer). Create a directory (mkdir $) and change to it (cd $). To be able to edit your website, you need to clone your site locally (save it to your computer). Type git clone and then paste the URL after. Boom! Your site is now ready to be expertly edited, with help by Yours Truly.

Let's make the front page of your website. Create a new file within your brand new cloned directory. To make a new index file (which most websites need), type touch index.html. You already have Sublime installed (but in case you dont, go here: here). You can edit index.html by opening Sublime. A shortcut to do that - type subl index.html. I don't know how to input HTML code and not have it show up, so I'll have to come back another time to show you how to write HTML code into your website. This is good enough for now, let's publish!

It took me a while to get this part down, but now I feel pretty proficient in this matter. To add index.html to GitHub, we need to add it! Makes sense, right? Type git add index.html. To make sure this was done correctly, type git status. This next part is the beautiful part of GitHub. In order to do anything, you need to tell your compatriots what you did! No more guess work, baby, everyone is accountable for what they contribute to the project. To tell everyone what you did, you need to open up the commit editor. Do this by typing "git commit -v". Sublime will open the file, and all you need to do is type "Add index.html". Why not "added index.html?" Because it's impolite, thats why. Once you do this, all you need to do is push the changes to the world wide web! Type git push to upload everything you've done to GitHub. Once this is done, go to your repository and make sure everything worked. 70% of the time, this has worked for me, 100% of the time. Surf's up! Catch some digital waves!
